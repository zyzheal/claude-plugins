### 自动触发的 Agent/Skill

| 触发时机 | 触发方式 | Agent/Skill | 说明 |
|----------|----------|-------------|------|
| `/project-init` | 命令触发 | Initializer Agent | 项目初始化 |
| `/requirement-dev` | 命令触发 | Planner Agent | 需求规划 |
| Planner 完成后 | 自动 | Coder Agent | 按 DAG 实现功能 |
| Coder 完成 | Hook (SubagentStop) | - | 自动处理审查/更新 |
| control_level=high | Hook 触发 | Reviewer Agent | 代码审查 |
| `/review-code` | 命令触发 | Reviewer Agent | 手动审查 |

## 4. content-create - 智能内容创作

### 工作流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ /content-create:write <topic> [platforms:wx,xhs] [skip-review:true]        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ Step 1: 初始化工作空间                                                       │
│ ├── 生成时间戳目录：article/YYYY-MM-DD-HH-MM/                              │
│ └── 读取配置文件                                                             │
│                                                                              │
│ Step 2: 数据收集 (collector agent)                                           │
│ ├── 从 RSS 源获取最新文章                                                    │
│ ├── Web 搜索补充热点信息                                                     │
│ └── 生成 hot_topic_data.md                                                   │
│                                                                              │
│ Step 3: 文章撰写 (writer-wx/writer-xhs agent)                                │
│ ├── 分析风格参考文章                                                        │
│ ├── 基于热点数据创作                                                        │
│ ├── 生成封面图和配图                                                        │
│ └── 输出 wx_article.md / xhs_article.md                                     │
│                                                                              │
│ Step 4: 质量评审 (reviewer agent) ← skip-review=true 时跳过                  │
│ ├── 风险合规检查                                                            │
│ ├── 内容质量评分 (0-100)                                                    │
│ ├── 模拟用户评论                                                            │
│ └── 如评分<85，提供修改建议并重新撰写 (最多 2 次)                               │
│                                                                              │
│ Step 5: 多平台发布                                                           │
│ ├── 公众号发布 (wenyan-mcp)                                                 │
│ ├── 小红书发布 (playwright)                                                 │
│ └── 飞书发布 (lark-mcp)                                                     │
│                                                                              │
│ Step 6: 生成总结报告                                                         │
│ └── workflow_summary.md                                                      │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

### 自动触发的 Agent

| 步骤 | Agent | 说明 |
|------|-------|------|
| Step 2 | collector | 热点信息收集 |
| Step 3 | writer-wx/writer-xhs | 公众号/小红书撰写 |
| Step 4 | reviewer | 文章质量评审 |
| Step 5 | - | 平台发布（MCP/自动化） |
