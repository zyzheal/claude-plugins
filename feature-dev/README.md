# feature-dev（功能开发）

结构化的 7 阶段功能开发工作流，通过专业 Agent 进行代码库探索、架构设计和质量审查，确保新功能与现有代码无缝集成。

## 设计理念

构建功能不仅仅是写代码，还需要：
- **理解代码库** 再进行变更
- **提出问题** 澄清模糊需求
- **深思熟虑地设计** 再实现
- **质量审查** 构建完成后

本插件将这些实践融入到 `/feature-dev` 命令的自动工作流中。

## 安装

```bash
/plugin install feature-dev@claude-plugins-official
```

### 手动安装

```bash
ln -s /path/to/feature-dev ~/.claude/plugins/local/feature-dev
```

## 使用说明

### `/feature-dev <功能描述>`

启动 7 阶段的引导式功能开发流程。

```bash
# 带功能描述
/feature-dev 添加 OAuth 用户认证

# 不带描述（交互式引导）
/feature-dev
```

## 7 阶段工作流

### 阶段 1：需求发现

**目标：** 理解需要构建什么

**执行内容：**
- 澄清功能需求
- 明确要解决的问题
- 识别约束和前提条件
- 总结理解并与你确认

### 阶段 2：代码库探索

**目标：** 理解相关的现有代码和模式

**执行内容：**
- 启动 2-3 个 `code-explorer` Agent 并行探索
- 每个 Agent 关注不同方面（类似功能、架构、UI 模式）
- 返回完整分析报告和关键文件列表
- Claude 阅读所有识别的文件以建立深入理解

### 阶段 3：澄清问题

**目标：** 填补空白，消除所有歧义

**执行内容：**
- 审查代码库发现和需求
- 识别未充分指定的方面：
  - 边界情况
  - 错误处理
  - 集成点
  - 向后兼容性
  - 性能需求
- 以组织列表呈现所有问题
- **等待你的回答后再继续**

### 阶段 4：架构设计

**目标：** 设计多种实现方案

**执行内容：**
- 启动 2-3 个 `code-architect` Agent，各有不同侧重点：
  - **最小变更**：最小改动，最大复用
  - **清晰架构**：可维护性，优雅的抽象
  - **务实平衡**：速度 + 质量
- 审查所有方案
- 形成最适合此任务的推荐
- 呈现对比和权衡
- **询问你偏好哪个方案**

### 阶段 5：实现

**目标：** 构建功能

**执行内容：**
- **等待明确批准**后才开始
- 阅读之前阶段识别的所有相关文件
- 按照选定的架构实现
- 严格遵守代码库约定
- 编写清晰、有文档的代码
- 持续更新进度

### 阶段 6：质量审查

**目标：** 确保代码简洁、DRY、优雅且功能正确

**执行内容：**
- 启动 3 个 `code-reviewer` Agent 并行审查：
  - **简洁/DRY/优雅**：代码质量和可维护性
  - **Bug/正确性**：功能正确性和逻辑错误
  - **约定/抽象**：项目标准和模式
- 合并发现
- 识别最高优先级问题
- **呈现结果并询问处理方式**

### 阶段 7：总结

**目标：** 记录已完成的工作

**执行内容：**
- 标记所有待办完成
- 总结：
  - 构建了什么
  - 关键决策
  - 修改的文件
  - 建议的后续步骤

## 内置 Agent

### `code-explorer`

**用途：** 深入分析现有代码库功能，追踪执行路径

**输出：**
- 入口点及文件:行引用
- 逐步执行流程
- 关键组件和职责
- 架构洞察
- 必读文件列表

### `code-architect`

**用途：** 设计功能架构和实施蓝图

**输出：**
- 发现的模式和约定
- 架构决策及理由
- 完整的组件设计
- 实施映射（具体文件）
- 分阶段的构建序列

### `code-reviewer`

**用途：** 审查代码的 Bug、质量问题和项目约定

**输出：**
- 关键问题（置信度 75-100）
- 重要问题（置信度 50-74）
- 具体修复建议（文件:行引用）
- 项目约定参考

## 使用模式

### 完整工作流（推荐用于新功能）

```bash
/feature-dev 为 API 端点添加速率限制
```

让工作流引导你完成所有 7 个阶段。

### 手动调用 Agent

```
# 探索功能
"启动 code-explorer 追踪认证机制"

# 设计架构
"启动 code-architect 设计缓存层"

# 审查代码
"启动 code-reviewer 检查最近的变更"
```

## 何时使用

✅ **适合：**
- 涉及多个文件的新功能
- 需要架构决策的功能
- 与现有代码的复杂集成
- 需求不太明确的功能

❌ **不适合：**
- 单行 Bug 修复
- 微小变更
- 定义明确的简单任务
- 紧急热修复

## 前置要求

- 已安装 Claude Code
- Git 仓库（用于代码审查）
- 有现有代码库的项目（工作流假设已有代码可学习）
