# playground（交互式游乐场）

创建自包含的交互式 HTML 游乐场——单文件探索工具，让用户通过控件可视化配置，实时预览，并复制生成的提示词回 Claude。

## 什么是游乐场？

游乐场是一个独立的 HTML 文件，包含：
- 一侧的交互式控件
- 另一侧的实时预览
- 底部的提示词输出和复制按钮

用户调整控件，可视化探索，然后将生成的提示词复制回 Claude。

## 安装

```bash
/plugin install playground@claude-plugins-official
```

### 手动安装

```bash
ln -s /path/to/playground ~/.claude/plugins/local/playground
```

## 何时使用

当用户请求交互式游乐场、探索器或可视化工具时使用——特别是当输入空间较大、可视化或结构化，难以用纯文本表达时。

## 模板

插件包含常见游乐场类型的模板：

| 模板 | 用途 |
|------|------|
| **design-playground** | 视觉设计决策（组件、布局、间距、颜色、排版） |
| **data-explorer** | 数据和查询构建（SQL、API、管道、正则表达式） |
| **concept-map** | 学习和探索（概念图、知识缺口、范围映射） |
| **document-critique** | 文档审查（带批准/拒绝/评论工作流的建议） |

## 使用说明

### 自然语言触发

```
"创建一个颜色配置游乐场"
"帮我做一个布局选择的可视化工具"
"我需要一个正则表达式测试器"
"做一个间距系统的交互式探索"
```

### 工作流程

1. Claude 根据需求选择合适的模板
2. 生成包含控件和预览的 HTML 文件
3. 用户调整控件探索不同配置
4. 复制生成的提示词回 Claude 继续工作

## 特性

- ✅ 单文件自包含，无需服务器
- ✅ 实时预览变更
- ✅ 一键复制生成的提示词
- ✅ 多种模板适配不同场景
- ✅ 美观的 UI 和交互体验

## 适用场景

✅ **适合：**
- 设计系统探索（颜色、排版、间距）
- 数据查询构建和测试
- 概念学习和知识图谱
- 文档审查和反馈
- 任何需要可视化配置的场景

❌ **不适合：**
- 简单的文本修改
- 不需要交互式的工作
- 纯代码生成任务
