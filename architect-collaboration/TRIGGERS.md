## Phase 1: 需求分析

### 触发短语

```
"分析需求"
"需求分析"
"需求收集"
"需求评审"
"澄清范围"
"可行性评估"
"这个需求的业务目标是什么"
```

### Skill 会自动

1. 询问业务目标和背景
2. 引导功能/非功能需求分解
3. 评估可行性
4. 验证完整性
5. 生成需求文档

### 输出

```markdown
docs/requirements.md - 需求文档
docs/risk-assessment.md - 风险评估
docs/stakeholder-analysis.md - 干系人分析
```

## Phase 2: 技术设计

### 触发短语

```
"设计方案"
"技术设计"
"架构设计"
"系统设计"
"技术方案"
"技术评审"
"为这个功能设计技术方案"
```

### Skill 会自动

1. 分析需求文档
2. 提出架构选项（含优缺点）
3. 引导 TDD 方法
4. 生成技术文档（含图表和伪代码）
5. 验证设计是否符合需求

### 输出

```markdown
docs/technical-design.md - 技术设计文档
docs/architecture-diagram.md - 架构图
docs/pseudocode.md - 伪代码
docs/test-strategy.md - 测试策略
```

## Phase 3: 任务拆解

### 触发短语

```
"拆解任务"
"任务分解"
"任务拆解"
"任务规划"
"任务评审"
"创建任务"
"把这个功能拆分成可执行的任务"
```

### Skill 会自动

1. 分析技术设计提取工作项
2. 拆分为粒度任务（≤2 人天）
3. 定义任务细节（输入/输出/依赖）
4. 建立优先级
5. 生成任务清单

### 输出

```markdown
docs/开发任务.md - 任务列表
docs/task-dependencies.md - 依赖映射
docs/sprint-plan.md - Sprint 计划
docs/estimation.md - 工时评估
```

## Phase 4: 功能开发

### 触发短语

```
"实现功能"
"功能开发"
"代码开发"
"编写代码"
"编写测试"
"TDD 开发"
"帮我实现这个功能"
```

### Skill 会自动

1. 阅读任务文档和开发任务.md
2. 引导 TDD 开发（测试先行）
3. 确保代码质量
4. 验证测试覆盖率（≥80%）
5. 更新进度

### 输出

```markdown

docs/progress-tracker.md - 进度跟踪
```

## 代码审查

### 触发短语

```
"审查代码"
"代码审查"
"代码评审"
"检查安全性"
"安全检查"
"性能审查"
"审查刚才提交的代码"
```

### Skill 会自动

1. 读取代码变更
2. 5 维度审查：
   - 安全性（25%）
   - 性能（20%）
   - 可维护性（25%）
   - 规范性（15%）
   - 测试（15%）
3. 问题分级（阻塞/严重/主要/次要）
4. 生成审查报告

### 输出

```markdown
docs/review-report.md - 审查报告
```

## 组合使用示例

### 完整项目流程

```
1. "分析这个需求：实现用户积分系统"
   → Requirements Analysis Skill
   → 产出：requirements.md

2. "为积分系统设计技术方案"
   → Technical Design Skill
   → 产出：technical-design.md

3. "拆解任务"
   → Task Breakdown Skill
   → 产出：开发任务.md

4. "帮我实现积分计算功能，用 TDD 方法"
   → Feature Development Skill
   → 产出：代码 + 测试

5. "审查刚才的代码"
   → Code Review Skill
   → 产出：review-report.md
```

## 新增需求评审

### 场景：项目中已有文档，需要评审新需求

```
用户："评审新需求：实现用户签到功能，先阅读现有需求文档"

Skill 会自动：
1. 读取 docs/requirements.md
2. 分析新需求与现有需求的一致性
3. 评估影响范围
4. 更新需求文档
```

## 技巧与注意事项

### ✅ 最佳实践

1. **明确上下文** - 提及具体功能或模块
   - 好："分析用户登录需求"
   - 差："分析需求"

2. **指定重点** - 说明关注的维度
   - "审查代码，重点关注安全性"
   - "设计方案，考虑高并发场景"

3. **提供背景** - 帮助 Skill 更好理解
   - "这是一个电商项目，需要支持双 11 流量"

### ⚠️ 常见问题

**Q: Skill 没有被触发？**
- 确保使用上述触发短语
- 添加更多上下文帮助匹配
- 检查插件是否正确安装

**Q: 输出不符合预期？**
- 提供更详细的需求描述
- 指定输出格式要求
- 使用多轮对话澄清
