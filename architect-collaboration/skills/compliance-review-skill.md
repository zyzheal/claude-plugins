# Compliance Review Skill - 合规专家评审技能

本技能提供专业的合规性评审能力，覆盖数据本地化、隐私保护 (GDPR/个人信息保护法)、审计要求等合规领域。

## 适用法规框架

### 1. GDPR (欧盟通用数据保护条例)

**适用范围**：
- 处理欧盟居民个人数据的组织
- 无论组织是否位于欧盟境内

**核心要求**：
```
1. 合法、公平、透明原则
   - 数据处理有合法依据
   - 数据处理目的明确
   - 向数据主体透明

2. 数据最小化
   - 仅收集必要数据
   - 不超范围使用

3. 准确性
   - 数据准确及时更新
   - 错误数据及时更正

4. 存储限制
   - 不超必要期限存储
   - 定义保留策略

5. 完整性和保密性
   - 适当安全措施
   - 加密/假名化
```

**数据主体权利**：
| 权利 | 说明 | 实现要求 |
|------|------|----------|
| 知情权 | 了解数据处理情况 | 隐私政策、数据访问接口 |
| 访问权 | 获取个人数据副本 | 数据导出功能 |
| 更正权 | 更正不准确数据 | 数据修改功能 |
| 删除权 (被遗忘) | 请求删除数据 | 数据删除功能 |
| 限制处理权 | 限制数据处理 | 暂停处理机制 |
| 数据可携权 | 获取可携带格式 | 结构化/通用格式导出 |
| 反对权 | 反对特定处理 | 退出机制 (如营销) |

---

### 2. 中国个人信息保护法 (PIPL)

**核心要求**：
```
1. 告知 - 同意原则
   - 明示处理目的、方式、范围
   - 取得个人同意
   - 单独同意 (敏感信息)

2. 最小必要原则
   - 具有明确、合理的目的
   - 与处理目的直接相关
   - 对个人权益影响最小

3. 分类分级保护
   - 一般个人信息
   - 敏感个人信息 (生物识别、医疗健康、金融账户等)
```

**跨境传输要求**：
| 条件 | 要求 |
|------|------|
| 告知同意 | 单独同意 + 告知境外接收方信息 |
| 影响评估 | 个人信息保护影响评估 |
| 标准合同 | 签订标准合同/通过认证 |
| 安全评估 | 向网信部门申报安全评估 |

---

### 3. 数据本地化要求

**中国数据本地化**：
```
关键信息基础设施运营者 (CIIO):
- 在中国境内运营中收集和产生的
- 个人信息和重要数据
- 应当在境内存储

确需向境外提供的:
- 按照国家网信部门会同国务院有关部门制定的办法
- 进行安全评估
```

**行业特定要求**：
| 行业 | 要求 |
|------|------|
| 金融 | 金融数据境内存储，跨境需审批 |
| 医疗 | 健康医疗数据境内存储 |
| 汽车 | 车主/驾驶人信息、地理信息境内存储 |
| 地图 | 地图数据、地理信息严格管控 |

### 2. 隐私保护设计 (Privacy by Design)

| 检查项 | 说明 | 合格标准 |
|--------|------|----------|
| **隐私政策** | 隐私政策完整性 | 覆盖所有数据处理场景 |
| **同意管理** | 同意收集和管理 | 明确、具体、可撤回 |
| **数据访问** | 数据主体权利实现 | 访问/更正/删除接口 |
| **数据保留** | 保留期限和删除 | 自动删除机制 |
| **DPO 指定** | 数据保护官 | 如需要则指定 |

**同意管理要求**：
```
✅ 有效的同意：
- 自由选择 (非捆绑)
- 具体明确 (按目的)
- 知情 (清晰说明)
- 明确行为 (勾选/点击)
- 可撤回 (与同意同样容易)

❌ 无效的同意：
- 默认勾选
- 捆绑同意
- 不明确表述
- 无法撤回
```

---

### 3. 审计合规

| 检查项 | 说明 | 合格标准 |
|--------|------|----------|
| **审计日志** | 数据处理活动日志 | 完整记录处理活动 |
| **日志保留** | 保留期限合规 | 满足法规要求 (如 3 年) |
| **审计追溯** | 可追溯数据处理 | 谁/何时/处理了什么 |
| **定期审计** | 审计机制 | 年度/专项审计 |

**审计日志要求**：
```
必须记录的事件：
- 个人信息的收集
- 个人信息的访问
- 个人信息的修改
- 个人信息的删除
- 个人信息的跨境传输
- 数据泄露事件

日志字段要求：
- 时间戳
- 操作者身份
- 操作类型
- 涉及数据范围
- 操作结果
- 操作原因 (如适用)
```

## 常见反模式

### 1. 隐私政策反模式

```
❌ 错误：隐私政策模糊
   "我们可能收集您的相关信息"→ 不具体

❌ 错误：捆绑同意
   "使用本服务即表示您同意所有条款"

❌ 错误：撤回困难
   同意容易，撤回需要联系客服

✅ 正确：清晰、具体、可操作
   明确说明收集什么、为什么、如何撤回
```

### 2. 数据保留反模式

```
❌ 错误：无限期保留
   "我们保留您的数据直到您要求删除"

❌ 错误：一刀切
   所有数据都保留相同期限

✅ 正确：分类定义保留期限
   账户数据：账户存续期 +3 年
   日志数据：6 个月
   交易数据：10 年 (法律要求)
```

### 3. 同意管理反模式

```
❌ 错误：默认勾选
   [✓] 我同意隐私政策 (默认勾选)

❌ 错误：黑暗模式
   用颜色/位置诱导用户同意

❌ 错误：无法细分
   要么全同意，要么全不同意

✅ 正确：明确、主动、可细分
   [ ] 我同意 (用户主动勾选)
   按目的分别同意
```
