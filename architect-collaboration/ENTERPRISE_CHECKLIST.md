# 企业级评审清单

> 基于 19 个专家角色的完整评审检查清单

---

## 快速索引

| 阶段 | 参与角色 | 说明 |
|------|----------|------|
| **Phase 1: 需求分析** | ①产品专家、③PMO、④首席架构师 | 需求完整性、可行性评审 |
| **Phase 2: 技术设计** | ④首席架构师、⑤前端架构师、⑥插件系统、⑦AI/算法、⑧数据库、⑨开源集成、⑪安全、⑫合规、⑬可观测性、⑭中间件、⑮SRE | 架构、安全、数据库、可观测性等 |
| **Phase 3: 任务拆解** | ③PMO、⑩测试专家、⑰DevOps | 任务规划、测试策略 |
| **Phase 4: 功能开发** | ⑩测试专家、⑪安全专家 | 代码审查、安全扫描 |
| **发布评审** | ①产品专家、⑩测试专家、⑪安全专家、⑫合规专家、⑮SRE、⑯运维、⑱技术文档 | 发布前检查 |
| **上线后评审** | ⑲技术委员会 | 技术债务、复盘 |

---

## 使用说明

### 触发方式

```
# 需求评审阶段
"进行需求评审，邀请产品专家、PMO、首席架构师参与"

# 架构评审阶段
"进行架构评审，邀请首席架构师、前端架构师、数据库专家、安全专家参与"

# 设计评审阶段
"进行设计评审，邀请前端架构师、插件系统专家、AI 专家参与"

# 开发评审阶段
"进行开发评审，邀请测试专家、安全专家参与"

# 发布评审阶段
"进行发布评审，邀请产品专家、测试专家、安全专家、合规专家、SRE、运维参与"

# 上线后评审阶段
"进行上线后评审，邀请技术委员会参与"
```
